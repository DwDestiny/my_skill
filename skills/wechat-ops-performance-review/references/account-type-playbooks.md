# 账号类型 Playbook — 类型体系、识别信号与差异化分析链路

> 本文是账号类型路由引擎（`scripts/analyze/m9_account_type.py`，engine `account-type-router-v1`）的方法论长文。
> SKILL.md 只放触发与路由摘要；判定阈值与打分细节以代码为准，本文解释"为什么这样分、每类怎么看"。

## 0. 为什么要分类型

同一份数据，不同类型的号该得出完全不同的结论：

- 资讯号阅读中位 800、评论率 0.2%，是**正常状态**；同样数字放在 IP 号身上是**人设失灵**。
- 转化号阅读中位偏低但漏斗通畅，是**健康**；按媒体号口径去"冲量"反而毁掉精准度。
- 知识号一堆"低读高享"的深度遗珠，问题在**标题**不在选题；按通用口径会误判成"内容不行"。

所以诊断链路必须先回答"你是个什么类型的号"，再决定"哪些指标要紧、结论怎么读、劲往哪使"。

## 1. 价值中立红线（与 DATA_CONTRACT §0.2 一致）

1. 类型之间**不判优劣**：媒体号、带货号、IP 号都是正当形态。
2. 类型判定是**诊断口径的路由**，不是给账号贴标签定性；判定结果附依据与置信档，置信不足回退通用链路，绝不硬判。
3. 所有可渲染文案不得出现"置信度"及百分比置信数字（配比百分比除外）。

## 2. 六大类型 + 通用回退

| key | 类型 | 一句话画像 |
|---|---|---|
| `media_news` | 媒体资讯号 | 快、多、轻：追时效、拼覆盖，阅读量与打开率是命 |
| `personal_ip` | 内容IP/个人品牌号 | 人味浓、认同强：靠判断与人设换信任，忠诚度是命 |
| `knowledge_service` | 知识服务/教育号 | 厚、体系化：教程方法清单，收藏与工具性价值是命 |
| `conversion_sales` | 私域转化/带货号 | 漏斗导向：内容为引流转化服务，链路通畅是命 |
| `brand_org` | 企业品牌号 | 机构口吻：服务业务与品牌，有效触达是命 |
| `local_life` | 本地生活号 | 地域绑定：同城粉丝浓度与线下转化是命 |
| `general` | 通用链路（回退） | 类型信号不清晰时的兜底，按通用诊断口径走 |

现实中大量账号是混合形态。引擎输出 `primary` + `secondary`（次优类型达阈值时），解读时以 primary 口径为主、secondary 作补充视角，不强行二选一。

## 3. 自动识别：信号 → 打分 → 置信 → 路由

### 3.1 识别信号（全部来自已抓取数据）

| 信号 | 数据来源 | 主要区分 |
|---|---|---|
| 发文频率 | `account_profile.publish_frequency` | 媒体号（高频）vs 知识/IP号（低频深耕） |
| 时效题材占比 | 标题/摘要正则（快讯/速递/今日/发布/上线…） | 媒体号 |
| 人味浓度 | 第一人称 + 个人叙事正则（我的/亲历/复盘/踩坑…） | IP号 vs 机构号 |
| 方法密度 | 教程类正则（步骤/拆解/清单/攻略/保姆级…） | 知识号 |
| 篇幅中位 | `article_length_chars` 中位 | 轻资讯 vs 厚方法 |
| 变现钩子密度 | 转化正则（扫码/加群/领取/付费/报名/下单…） | 转化号 |
| 促销词占比 | 福利正则（免费/优惠/限时/秒杀…） | 转化号辅助 |
| 机构口吻 | 官方口吻正则（我司/官方/旗下/荣获/喜报…） | 品牌号 |
| 本地词密度 | 本地正则（探店/门店/同城/商圈/开业…） | 本地号 |
| 互动结构 | share/comment/like 率的相对高低 | 认同型 vs 传播型 |
| 粉丝城市集中度 | `modules.audience.city` 头部城市占比 | 本地号 |

刻意的正则边界：时效词表**不含**"限时/福利"（那是促销词），避免转化号被误判为媒体号。

### 3.2 打分与置信

- 每类型按加权特征命中打 0–1 分（权重与阈值见 `m9_account_type.py._score_types`）。
- 置信规则：
  - 稳定样本 < 8 篇 → 直接回退 `general`（low）。
  - 最高分 < 0.45 → 回退 `general`（low），不硬判。
  - 最高分 ≥ 0.6 且与次优拉开 ≥ 0.15 → high。
  - 其余 → medium（判定成立，建议积累样本后复核）。
- 判定结果、依据（引用真实特征值）、全类型得分、决策说明全部写入 `report.json` 的 `account_type` 顶层节点（只增不删，向后兼容）。

### 3.3 路由机制

每类型的 playbook 定义 m1–m8 八个模块的权重（高/中/低），`routing.chain` 按权重排出诊断阅读顺序。**低权重模块照常产出**，差异化体现在：

- `reading_guide`：该类型的解读口径（哪些指标要紧、哪些不必焦虑）。
- `action_bias`：行动建议的倾斜方向（什么优先做、什么降级）。
- Markdown 报告新增「账号类型与分析路由」一节，把口径差异直接写给运营者看。

## 4. 分类型 Playbook 详解

### 4.1 媒体资讯号（media_news）

**核心成功指标**：打开率与阅读量、时效命中率（热点跟进速度）、发文频率稳定性。

**为什么**：资讯号的商业模型是覆盖面（流量主分成/广告），内容生命周期以小时计。断更一天就掉量，热点晚跟半天就没肉吃。

**模块权重**：checkup 高 / viral_genes 高 / content_engine 高 / audience 低 / growth_funnel 中 / action_plan 高 / standards 中 / forward 中。

**解读口径**：
- 阅读中位、发文频率、热点响应窗口 > 互动率。评论率低是形态使然，不是病。
- 分享率是破圈信号，值得盯；在看率参考价值低。
- 爆款基因模块权重高：资讯号的爆款公式（题材×标题型×时段）复制价值最大。

**行动倾斜**：提频提速 > 标题钩子实验 > 互动动作（后置）。

**常见误诊**：拿 IP 号口径批评"没有人设""互动差"——资讯号根本不靠这个活。

### 4.2 内容IP/个人品牌号（personal_ip）

**核心成功指标**：在看率与分享率（认同强度）、粉丝忠诚与复访、人设一致性。

**为什么**：IP 号卖的是"这个人的判断"，信任是唯一资产。阅读量可以低，认同不能散。

**模块权重**：audience 高 / growth_funnel 高 / action_plan 高 / forward 高 / checkup 中 / viral_genes 中 / content_engine 中 / standards 低。

**解读口径**：
- 在看率、评论质量、净增的口碑成分 > 阅读中位。
- 低读高在看的文章是**资产**不是失败。
- 题材漂移是头号风险信号：content_engine 里杂题材占比升高要报警。
- 量化标准（standards）权重低：IP 号不该拿"本号均值 1.5 倍"逼自己追爆款。

**行动倾斜**：巩固人设主线 > 高认同内容系列化 > 追热点（只追人设强相关）。

### 4.3 知识服务/教育号（knowledge_service）

**核心成功指标**：收藏与分享率（工具性价值）、方法内容密度、教程系列完成度。

**为什么**：知识号的内容是"存着用"的，生命周期以月计。单篇爆不爆不要紧，体系立不立得住才要紧。

**模块权重**：viral_genes 高 / content_engine 高 / standards 高 / action_plan 高 / checkup 中 / audience 中 / growth_funnel 中 / forward 中。

**解读口径**：
- 四象限里"深度遗珠"（低读高享）占比高 = 标题拖后腿，不是内容问题。viral_genes 的 `diagnosis_signal.title_problem` 对知识号权重最高。
- 分享率与长文完读 > 阅读峰值。
- 发文频率可以低，但厚度和稳定性不能降。

**行动倾斜**：深度遗珠标题重做 + 存量二次分发 > 散篇整合成系列 > 提频（最后考虑）。

### 4.4 私域转化/带货号（conversion_sales）

**核心成功指标**：引流-转化漏斗通畅度、私域承接动作密度、转化内容配比。

**为什么**：转化号的公众号只是漏斗上半段，最终收入在私域/成交。单篇阅读量是虚荣指标，链路断点才是病灶。

**模块权重**：checkup 高 / content_engine 高 / audience 高 / growth_funnel 高 / action_plan 高 / viral_genes 中 / forward 中 / standards 低。

**解读口径**：
- 拉新篇看阅读、信任篇看互动、转化篇看钩子密度——**分开评估**，不能拿同一把尺子。
- 阅读中位偏低但漏斗通畅 = 健康；盲目冲量反而稀释精准度。
- 转化钩子密度过高会反噬阅读（全是广告没人看），content_engine 的配比诊断要盯这个平衡。
- 粉丝画像与来源分析权重拉高：精准比规模重要。

**行动倾斜**：补漏斗断点 > 控钩子密度 > 画像精准化。

### 4.5 企业品牌号（brand_org）

**核心成功指标**：品牌内容与业务关联度、发布稳定性、有效触达（非僵尸阅读）。

**为什么**：品牌号不靠内容变现，靠内容维护品牌资产与客户触达。最大的病是自嗨——只有官宣没有用户价值。

**模块权重**：checkup 高 / content_engine 高 / audience 高 / action_plan 高 / growth_funnel 中 / standards 中 / viral_genes 低 / forward 低。

**解读口径**：
- 互动全低时先区分：触达失败（没人看到）还是内容无关（看到不想理）。
- 爆款公式参考价值低：品牌号不该追平台热点爆款。
- 阅读构成要拆：员工转发撑起来的阅读不算有效触达。

**行动倾斜**：官宣改写为用户价值视角 > 固定栏目化 > 稳定发布；爆款复制类动作降级。

### 4.6 本地生活号（local_life）

**核心成功指标**：本地粉丝浓度（城市集中度）、到店/线下转化钩子、本地题材覆盖密度。

**为什么**：本地号的价值密度在地理半径内。一万泛粉不如三千同城粉。

**模块权重**：content_engine 高 / audience 高 / growth_funnel 高 / action_plan 高 / checkup 中 / viral_genes 中 / forward 中 / standards 低。

**解读口径**：
- 粉丝画像的城市分布是**第一诊断指标**（audience 模块对本地号权重最高）。
- 阅读量要结合本地人口基数评估，不用平台通用基准。
- 跨城传播的"爆款"对本地号可能是负资产（拉低粉丝浓度）。

**行动倾斜**：本地强绑定选题 > 每篇带线下动作钩子 > 泛流量内容降配比。

### 4.7 通用链路（general，回退）

触发条件：样本 < 8 篇，或所有类型得分 < 0.45。

**口径**：按原有通用诊断走（checkup/viral_genes/content_engine/action_plan 高权重），不预设类型化打法；样本与信号积累后引擎自动切换。回退不是失败，是"现在下类型结论会误导你"的诚实表达。

## 5. 已知边界与演进方向

- 识别基于标题+摘要正则近似，正文覆盖率低时信号偏保守（宁回退勿硬判）。
- 知识付费号与知识服务号暂不细分（变现痕迹 + 方法密度同时高时，primary/secondary 会同时给出 knowledge_service 与 conversion_sales，解读时合并看）。
- 粉丝增长模式（关注来源结构）在 audience 数据可用时可加入信号，当前为可选增强。
- 类型化的差异目前落在解读口径与行动倾斜层；后续可让 m6 action_plan 的 lane 分配直接消费 module_weights（需先在 DATA_CONTRACT 记录再动）。
