# 公众号运营看板设计系统

## 2026-07 高级简约化改版 · 设计意图

**视觉方向**:「编辑感体检报告」。奶白底不变、马卡龙语义色不变,但把气质从"数据面板"推向"精装刊物":判断句用**中文衬线**(Noto Serif SC / Songti SC),数据用 Space Grotesk,说明用 PingFang。衬线只给"判断"用——读者扫一眼就能凭字体分辨"这是结论"还是"这是证据",这是本次改版的默会引导核心。

**字阶系统(6 级)**:hero 判断 `clamp(34-46px)/serif/600` → 屏级结论 deck `clamp(24-30px)/serif/600` → 图表标题/角色 `15-16px/600` → 正文 `15px/400` → 标签 `13px` → 口径 caption `12px`。全站只有概览 verdict 一处 hero 级,每屏只有一处 deck 级——大字稀缺,层级才成立。

**注意力动线(F1→F2→F3)**:
- **F1** 打开即见:概览首屏衬线大字总判断(verdict)+ 一行 `→ next_action`。
- **F2** 视线下落:健康分环 + 关键数字带(hairline 分隔,Space Grotesk),证明判断有据。
- **F3** 首屏收尾:爆款基因卡(signature),把"规律"变成可记住的配方,钩住读者继续下滑。
- 之后整条滚动动线复现思考链:**结论(概览)→ 证据(01-06:体检/爆款/内容/读者/涨粉/标准)→ 行动(照镜子 → 找方向 → 规划矩阵 → 本周行动)**。幕与幕之间用"幕间屏"过渡——一句衬线提问(「为什么是这个判断?」「那接下来怎么办?」)+ 大量留白,让读者自己"问出"下一幕,不用步骤说教。

**栅格/留白策略**:主列 `max-width 1020px`;文字块(结论/说明/注)收窄到 `~660px` 阅读宽,图表铺满主列——宽窄交替本身就是错落节奏。屏间距靠 `min-height:100vh + scroll-snap` 一屏一事;屏内区距 40-56px,焦点周围留白最多。

**卡片/图表尺寸节奏**(信息重要性 = 视觉面积):
- 概览:全幅 verdict(大)→ 数字带(中)→ 基因卡(全幅但轻,细渐变描边)。
- 四象限散点(signature 数据屏):`1fr + 180px` 主图配窄图例,高 380px,全站最大图。
- 题材排行:按中位阅读降序的排行条,第一名自然最长,排行即层级。
- 读者画像:`1.1fr / 1fr` 双列;涨粉:全幅曲线(300px)+ 四周横向时间轴。
- 量化标准:一个 64px 巨数 + 右列三行小注(刻意的 62/38)。
- 行动屏:三篮子按置信做**重量差**——「现在就做」大字强色、「小步验证」正常、「暂不拍板」缩小降灰,置信度内化为版面响度,不出现任何分数。

**信息密度**:每屏一个核心问题,deck 只放一句判断;数据观察(analysis)降为 deck 下的一行小字依据;下一步(action)固定在屏底单行。图表墙、并列等宽卡一律不做。

## Visual Philosophy

浅色 + 马卡龙粉彩：**活泼、明快、高级**。奶白底铺满视口（不做"灰框包白卡"的廉价外壳），多个低饱和粉彩色（薄荷绿、蜜桃橙、奶油黄…）作为分类与数据的语义色，让看板有呼吸感和生气，却不花哨。

这是"内容创作者的数据洞察工具"——它该像一份精致、明快、专属定制的体检报告，第一眼就让人觉得"这是冲我的号做的"。

**第一屏是论点（Hero = 公众号身份）**：开场不是抽象结论，而是这个号的**身份与数据画像**——头像、名称、核心数据（粉丝、总阅读、篇均、发文频率…）。先惊艳，证明"深入分析过这个账号"，再逐步引入干货。

## Colors

奶白 `canvas` 铺满，`surface` 白卡浮于其上但**不靠灰色外框分隔**——靠柔和暖阴影和留白。马卡龙色板轮换用于：内容分类（每类一色）、KPI 图标容器、图表系列。主强调用 `mint` 薄荷绿（active/主按钮/首选图表）。`danger` 用马卡龙珊瑚 `#FF7A8A`。

**克制**：粉彩多但每个区块主导一色，不堆砌。大面积留白让色彩透气。

**红线**：页面不出现"置信度/N%"——置信度通过视觉层级（emphasis）与措辞（voice）隐性传达。

## Typography

中文 PingFang/Noto Sans SC + 拉丁 Inter。`display` 用于公众号名和整体判断（大而重，字间距收紧显个性）。`metric` 用于画像大数字。**`conclusion-text` 是关键**：结论文字 17px/600，用 `on-surface` 主色（**不弱化成灰**），与图表一主一辅、相辅相成。

## Spacing & Layout

8px 基准。卡片 padding 24px，区块间距 28–36px。**左右两栏 `240px 侧栏 / 1fr 主区`**（去掉旧的右侧上下文栏，证据/下一步下沉进各模块卡内）。**奶白底铺满整个视口，去掉外层灰色边框**。主区 = 概览首屏 + 七模块下钻屏，scroll-snap 一屏一结论。

## Components

**Hero · 公众号身份画像（第一屏，惊艳）**：
- 头像（圆形，爬取的真实公众号头像）+ 名称（display 大字）+ 平台/周期 caption。
- **核心数据画像条**：粉丝数 / 总阅读 / 篇均阅读 / 发文数 / 发布频率 / 爆款数——每个是 metric 大数字 + label，图标容器用不同马卡龙色。
- 下方接整体判断 verdict（一句核心判断）。
- 这一屏的任务是"惊艳 + 证明深入分析"，不堆干货。

**KPI / 画像卡**：圆形马卡龙色图标容器（48px，不同指标不同色）+ label + metric 大数字 + 数据口径 caption（说明这个数字是什么，如"近30天篇均"）。**不显示捏造的 delta**——无对比期就不显示涨跌。

**图文相辅相成（关键，纠正旧版"文被弱化成灰侧栏"）**：
- 每个维度屏：**图为主视觉**（左/中），**结论文字为辅但不弱化**（17px/600 主色，紧邻图，有"①看图 → ②看结论"的视觉引导，如序号/连接线/箭头）。
- 右栏作为"持续上下文"可保留观察+下一步，但维度屏内的结论必须与图并列、可读、不灰化。

**图表（数据必须自解释）**：
- 每个图表/数字**标注清楚口径**：轴标题、单位、"中位阅读"等，杜绝裸数字（如标题结构的 290/450 必须标"中位阅读"）。
- 平滑 `monotone` 折线、马卡龙色条形（同组同色或分类多色）、深色圆角 tooltip。

**多模块排版（纠正 7 模块 4+3 难看）**：
- 同类模块（如标题结构 7 个 pattern）用**均匀自适应网格**（`auto-fit, minmax`），让每行模块数自然均衡，不要手动 4+3 错位。

**结论卡 emphasis（置信度视觉内化）**：
- `hero`：大字号、马卡龙强调色描边、主视觉位；
- `primary`：正常；
- `secondary`：低置信——适度缩小 + 降一档马卡龙色饱和，但**仍可读、不灰死**（弱化是"次要"，不是"消失"）。

**避免**：灰色外框、拥挤首屏、把结论文字弱化成灰。

## Layout & Composition（排版节奏 —— 错落有致，拒绝豆腐块）

**第一原则：每屏一个视觉焦点，大小错落，主次分明。绝不把模块均匀码成等宽等高的豆腐块**——那样平、闷、不高级。借鉴 bento grid 与编辑式排版：焦点块大而醒目，辅助块小而克制，留白有呼吸。把"最该看的"做到最大/最强调，次要信息做小、做淡、靠边。

### 不同数量的块怎么摆（关键）
- **1 块**：占主、大，作焦点（如单一大图表/单一大数字）。
- **2 块**：主辅黄金比（约 62 : 38），一大一小，**绝不 50:50 平分**。如"大图 + 窄结论栏"。
- **3 块**：一主两辅——一个大焦点 + 两个小块（叠放或并排）；或高低阶梯错落。
- **4 块**：bento——`1 大 + 3 小`（大块占角，小块 L 型环绕）或 `2 大 2 小` 对角；**不要 2×2 等分**。
- **多块（5+，如 7 个标题 pattern）**：**排行式**——按数值排序，最高的条最长/最大，依次递减，像排行榜；或"主图 + 侧边小条列表"。**绝不均匀网格**。

### 通用节奏
- 每屏先定焦点，给它最大/最强调；次要信息小、淡、靠边。
- 卡片大小**有意做出差异**，避免清一色等宽等高。
- 区块间距 28–36px，留白充足；焦点周围留更多白。
- 一处出彩（signature）：让最亮眼的那个数字/图成为这一屏的记忆点。

## Information Architecture · 三幕叙事（结论 → 为什么 → 怎么办）

页面顺序复现完整思考链路:**先给体检结论(诊断)→ 再展开六份证据(归因)→ 最后收到方向与行动(行动)**。幕与幕之间插「幕间屏」:一句衬线提问 + 留白,让读者自己"问出"下一幕。左栏导航同构分三组,组名即思考步骤(「为什么」「怎么办」),编号 01-06 编码证据的诊断逻辑顺序(不是装饰)。

## Signature · 爆款基因卡

整个看板的记忆点：把"爆款公式"（题材 ╳ 标题模式 ╳ 时段）呈现成一张**基因密码卡**——三因子用 ╳ 连接，马卡龙渐变描边，载入时三因子依次"拼接"成型（像基因组装）。它把抽象的"爆款规律"变成一眼可记、可复制的配方。样本不足（reliable=false）时，卡片转"线索"措辞 + 降饱和，不下死结论。02 爆款基因屏的四象限散点是它的数据展开。

## Motion · 高级动效（克制编排，尊重 reduced-motion）

- **载入序列**：概览首屏元素 stagger 淡入上浮（~120ms 间隔），不是全部同时弹出。
- **数字滚动**：健康分/粉丝数 count-up（rAF，~600ms ease-out）。
- **爆款基因卡**：三因子依次拼接，每个 ╳ 连接处微弹。
- **四象限散点**：点按象限 stagger 弹入，hover 放大 + 显标题。
- **滚动揭示**：七模块下钻屏 IntersectionObserver 触发淡入（进入视口才动）。
- **微交互**：卡片 hover 抬升 2px + 阴影加深；导航 active 平滑高亮。
- **技术**：原生 CSS transition/keyframes + IntersectionObserver + rAF，零重依赖，体积最优；`prefers-reduced-motion` 下全部降级为无动画。
